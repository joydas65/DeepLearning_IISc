# L01: GenAI and NLP Introduction

## 1. Course focus
The course focuses on NLP, GenAI, and Agentic AI.

## 2. AI vs AI hype
AI should be evaluated by asking:
- Does the task require human-like effort or training?
- Was the behavior directly coded or learned/search-based?
- Can the system make flexible/adaptive decisions?

## 3. Predictive AI vs Generative AI

| Point | Predictive AI | Generative AI |
|---|---|---|
| Input | Any data modality | Any data modality |
| Output | Continuous or categorical | Text, image, video, audio |
| Examples | Regression, classification | Text generation, image generation, code generation |
| Evaluation | Accuracy, F1, RMSE | Correctness, relevance, faithfulness, safety |

## 4. Foundation model
A foundation model is a large-scale model trained on diverse data and used as a base for downstream tasks.

Key capabilities:
- Prompt engineering
- RAG
- Fine-tuning
- Generalization to new tasks

Examples:
- GPT
- BERT
- T5

## 5. Agentic AI
Agentic AI systems can pursue goals, make decisions, and take actions in a domain.

Key aspects:
- State awareness
- Reactive and proactive behavior
- Access to tools/data

## 6. GenAI customer support architecture
Important components:
- Intent detection
- RAG
- Guardrails
- Text-to-SQL/tools
- Memory buffers
- Human handoff
- Feedback collection

## 7. AI/ML workflow
1. Frame the business problem
2. Gather and clean data
3. Establish baselines
4. Explore models
5. Evaluate and tabulate metrics
6. Deploy using MLOps/AIOps/LLMOps
7. Monitor and retrain

## 8. Exam traps
- RAG does not change model weights.
- Prompting is not fine-tuning.
- GenAI output can be fluent but wrong.
- Predictive AI outputs continuous/categorical values.
- GenAI outputs text/image/video/audio.
- Agentic AI is more than a chatbot.
